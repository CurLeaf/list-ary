# Listary 工具集 — 优化方案与新想法

> 基于完整代码库审查生成 | 2026-03-01  
> 设计原则：**轻便上手** / **性能克制** / **极客像素风** / **架构清晰**

## 设计原则

### 功能：轻便、低门槛
- 不堆砌无关功能，每个 Tab 只解决一个问题
- 新功能必须满足"打开即用"——零配置、零学习成本
- 拒绝重造轮子（不做剪贴板管理器、不做 Task Manager、不做 Notepad）

### 性能：克制、无驻留负担
- 后台零轮询，WebSocket 事件驱动
- psutil 调用加 TTL 缓存，避免重复系统调用
- SQLite 连接复用，不每次 new
- 单 EXE < 35MB，内存 < 80MB

### 样式：极客像素风
- 抛弃毛玻璃 glassmorphism，转向**像素风 / 终端美学**
- 等宽字体为主体（JetBrains Mono / Fira Code）
- 配色参考终端：深黑底 + 荧光绿/琥珀色/青色高亮
- 卡片边框用 1px 实线而非模糊阴影
- 按钮方正（小圆角 4px），不要大圆角胶囊
- 状态用 ASCII 字符（`[OK]` `[!!]` `[--]`）代替 emoji
- 动效极简：只保留 fade 0.1s，去掉 scale/slide/bounce
- 图标可考虑像素风 icon set 或纯 ASCII 符号

### 架构：职责清晰
- 当前问题：模块间有重复代码、import 时副作用、magic number 散落
- 目标结构见下方 [架构重构](#架构重构) 章节

## Bug 修复

### B1. 端口冲突无提示
**文件：** `hub.py → start_server()`  
**现象：** 9000 端口被占 → uvicorn 静默失败 → pywebview 白屏  
**修复：** 启动前 `socket.connect_ex` 检测，被占用则自动 +1 递增到 9005，仍失败则弹 messagebox

### B2. ssh_connect.py 模块级副作用
**文件：** `modules/ssh_connect.py:11`  
**现象：** `SERVERS_FILE = get_config_path(...)` 在 import 时求值，data 目录可能还没建  
**修复：** 改为惰性函数 `_get_servers_file()`

### B3. 上报状态丢失
**文件：** `dashboard/session_manager.py:89-95`  
**现象：** `completed` / `partial` / `blocked` 全部映射为 `waiting`，看板无法区分  
**修复：** 保留 `completed`、`blocked` 为独立状态，只把 `partial` 映射为 `waiting`

### B4. WebSocket 重连风暴
**文件：** `panel.html → connectWS()`  
**现象：** 断开后 1s 无脑重连，服务挂了会刷屏  
**修复：** 指数退避 `delay = Math.min(delay * 1.5, 30000)`，成功连接后重置

### B5. report.py 无限阻塞
**文件：** `report/report.py:228`  
**现象：** 用户不回复 → Windsurf 进程永远卡住  
**修复：** 加 `--max-wait` 参数（默认 3600s），超时输出 `[TIMEOUT]`

### B6. kill-port API 无类型校验
**文件：** `dashboard/server.py:243`  
**现象：** 传 `{"port": "abc"}` → 500 崩溃  
**修复：** 用 Pydantic `KillPortRequest(BaseModel)` 替代裸 dict

## 架构重构

### A1. 目标目录结构

```
list-ary/
  hub.py                  # 入口：CLI 分发 + GUI 启动
  config.py               # 所有常量 & 用户配置加载（NEW）
  utils.py                # 纯工具函数（路径、编码）
  modules/
    kill_port.py           # 杀端口（纯逻辑，不含 Rich UI）
    ssh_manager.py         # SSH 配置 CRUD（唯一数据源）
    ssh_connect.py         # SSH 连接（调用 ssh_manager.load_servers）
    windsurf_project.py    # 合并 setup + open（RENAME）
  dashboard/
    server.py              # FastAPI 路由
    session_manager.py     # SQLite 会话 CRUD
    models.py              # Pydantic 模型
    templates/
      panel.html           # GUI 唯一页面
  report/
    report.py              # 注入到项目的上报脚本
```

### A2. 新增 `config.py` — 统一常量

```python
# config.py
import os, json
from utils import get_data_dir

PORT = 9000
STUCK_TIMEOUT_MINUTES = 5
WINDOW_WIDTH = 1000
WINDOW_HEIGHT = 700
SERVER_READY_TIMEOUT = 6  # 秒

def load_settings() -> dict:
    p = os.path.join(get_data_dir(), "settings.json")
    if os.path.exists(p):
        with open(p) as f: return json.load(f)
    return {}
```

所有模块从 `config.py` 读常量，不再各自硬编码。

### A3. 消除重复代码

| 重复 | 位置 | 合并方案 |
|------|------|----------|
| `load_projects()` | `windsurf_open.py` + `windsurf_setup.py` | 合并为 `windsurf_project.py`，单文件导出 |
| `load_servers()` | `ssh_connect.py` + `ssh_manager.py` | `ssh_connect.py` 直接 `from .ssh_manager import load_servers` |
| 数据目录计算 | `session_manager.py` 自算 vs `utils.py` | 已修复（统一用 `get_data_dir()`） |

### A4. 结构化日志

**现状：** 所有错误 `except: pass` 或 `print` 到 stderr（EXE 无控制台看不到）  
**方案：**
```python
# utils.py
import logging
log = logging.getLogger("listary")
# 写到 %APPDATA%/ListaryTools/listary.log，最大 2MB 滚动
```
各模块统一 `from utils import log`，替换所有 `print(..., file=sys.stderr)` 和裸 `pass`。

### A5. API 请求模型补全

将 `server.py` 中所有裸 `dict` 参数换成 Pydantic 模型：

```python
# dashboard/models.py 新增
class KillPortRequest(BaseModel):
    port: int = Field(gt=0, le=65535)

class ServerConfig(BaseModel):
    name: str
    host: str
    port: int = 22
    path: str = ""
    key: str = ""

class ProjectSetup(BaseModel):
    path: str
    name: str
```

## 样式改版：像素风

### 整体方向

从当前的 **Glassmorphism（毛玻璃 + 大圆角 + 渐变）** 转向 **Terminal Geek（终端极客 + 像素感 + 方正）**。

### 配色方案

```
背景      #0c0c0c（纯黑偏暖）
卡片底    #141414（微亮灰黑）
边框      #2a2a2a（1px 实线，不模糊）
文字主    #e0e0e0（柔白）
文字次    #666666
主色      #00ff9f（荧光绿——成功/主操作）
警告      #ffb800（琥珀）
危险      #ff4444（红）
信息      #00bfff（青蓝）
选中高亮  #1a3a2a（绿底暗色块）
```

### 字体

- **主字体：** `JetBrains Mono`（全局等宽，极客感拉满）
- **回退：** `Fira Code, Consolas, monospace`
- 不再使用 Inter 等比例字体

### 组件规范

| 组件 | 当前（毛玻璃） | 改为（像素风） |
|------|----------------|----------------|
| 卡片 | `backdrop-filter:blur(20px)` + 大圆角 16px | `border:1px solid #2a2a2a` + 小圆角 4px + 无模糊 |
| 按钮 | 渐变 gradient + 阴影 | 纯色填充 `#00ff9f` + 黑色文字 + hover 反转 |
| 输入框 | 半透明底 + 圆角 10px | `border:1px solid #2a2a2a` + 黑底 + 圆角 2px |
| 导航 | 左竖条 3px indigo + 微亮背景 | 选中项前缀 `>` 符号 + 荧光绿文字 |
| Toast | 毛玻璃 + 左色条 + 右滑入 | 纯黑底 + 左侧 `[OK]`/`[!!]` 前缀 + fade 0.1s |
| Modal | scale(0.95) 缩放动画 | 无动画，直接显示 + 1px 边框 |
| 状态点 | 彩色圆点 + glow + pulse | ASCII 标记 `[OK]` `[..]` `[!!]` `[XX]` `[--]` |
| 图标 | Lucide 线性图标 | 可保留 Lucide（stroke-width 1px 更锐利）或换纯 ASCII |

### 侧边栏示例

```
  LISTARY v1.0
  ──────────────
  > KILL PORT      ← 选中：荧光绿 + > 前缀
    SSH CONNECT
    SSH CONFIG
    WINDSURF
    DASHBOARD
    SETTINGS
```

### 动效规则

- **只用 `opacity 0→1, 0.1s ease`**
- 删除所有 `translateY` / `scale` / `translateX` / `bounce`
- hover 效果：边框色变亮 `#2a2a2a → #444`，不位移不缩放
- active 效果：背景色反转（绿底黑字 → 黑底绿字）

### 技术栈不变

TailwindCSS CDN + Alpine.js + Lucide（或去掉换 ASCII）+ 单 HTML 零构建

## 功能优化

> 只打磨现有功能，不新增 Tab。保持 5 个核心 Tab：  
> **杀端口 / SSH 连接 / SSH 配置 / Windsurf 项目 / 调度看板**  
> 可选第 6 个：**设置**

### F1. 设置页（第 6 个 Tab）
轻量设置面板，只包含必要项：
- 服务端口（默认 9000）
- 卡死超时（默认 5 分钟）
- Windsurf 路径（自动检测 + 手动覆盖）
- 开机自启开关

保存到 `%APPDATA%/ListaryTools/settings.json`，重启生效。

### F2. 会话过滤
看板 Tab 顶部加 4 个状态过滤按钮（ASCII 风格）：
```
[ALL: 12]  [WAIT: 3]  [EXEC: 5]  [STUCK: 1]
```
纯前端 Alpine.js computed filter，零后端改动。

### F3. 会话自动清理
`stuck_checker` 协程中增加：7 天前的已完成/已取消会话自动删除。  
避免 SQLite 无限膨胀，内存稳定。

### F4. 系统托盘
关闭窗口 → 隐藏到托盘（不退出），后台服务保持运行。  
托盘右键菜单：`显示面板` / `退出`  
依赖：`pystray` + `Pillow`（生成一个 16x16 像素 icon）

### F5. 服务器状态探测
SSH 列表每个服务器旁显示 `[ON]` / `[--]`：
- 后台每 60s TCP 探测一次（`socket.connect_ex`，timeout=1s）
- 通过 WebSocket 推送状态
- 不阻塞 UI，不消耗额外内存

### F6. 项目支持编辑
Windsurf 项目列表加"编辑"按钮，可修改项目名称。  
复用现有弹窗组件，不增加新 UI 模式。

### F7. 端口列表搜索
杀端口页的活跃端口列表顶部加一个搜索框：
```
[_search________________]
```
Alpine.js 前端过滤，匹配端口号或进程名。零后端改动。

## 性能优化

### P1. SQLite 连接复用
**现状：** 每个 API 调用 `sqlite3.connect()` 新建连接  
**方案：** `threading.local()` 线程级单例：
```python
_local = threading.local()
def get_conn():
    if not hasattr(_local, 'conn'):
        _local.conn = sqlite3.connect(DB_PATH, check_same_thread=False)
    return _local.conn
```

### P2. psutil 结果缓存
**现状：** `net_connections()` 每次调用 ~100ms  
**方案：** 1 秒 TTL 缓存，同一秒内多次请求复用结果

### P3. WebSocket 增量推送
**现状：** 任何 session 变更都广播全量列表  
**方案：** 广播单条变更 `{"type":"update", "session":{...}}`，前端按 `session_id` patch

## 不做清单

以下想法经评估**不符合轻便原则**，明确不做：

| 想法 | 不做理由 |
|------|----------|
| 剪贴板历史管理器 | 功能膨胀，偏离核心定位，有专门工具（Ditto） |
| 进程管理器 Tab | 重造 Task Manager，杀端口已够用 |
| 环境变量查看器 | 偏离核心，`set` 命令即可 |
| 快速笔记 / Scratch | 偏离核心，有 Notepad / Obsidian |
| 多看板支持 | 过度设计，当前单机场景够用 |
| 项目模板系统 | 过度设计，两个 workflow 已覆盖主流场景 |
| Git Hook 自动上报 | 侵入性强，与 Windsurf 工作流冲突 |
| Session 历史导出 | 低频需求，可直接查 SQLite |
| 全局快捷键 | 需额外依赖（keyboard/pynput），增加内存驻留 |
